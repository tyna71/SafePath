# Contributing to SafePath

Thanks for your interest in SafePath! Contributions of any size are welcome —
bug reports, feature ideas, documentation fixes, or code.

## Reporting a bug

Open an [issue](../../issues) and include:
- What you expected to happen vs. what actually happened
- Steps to reproduce
- Screenshots if it's a UI issue
- Device/platform (Android, web, etc.)

## Suggesting a feature

Open an issue describing:
- The problem it solves
- Roughly how you imagine it working

## Submitting changes

1. Fork the repo
2. Create a branch: `git checkout -b feature/your-feature-name`
3. Make your changes
4. Commit with a clear message: `git commit -m "Add: short description"`
5. Push and open a Pull Request describing what changed and why

## Code style

- Follow the existing Flutter/Dart formatting (`dart format .`)
- Keep PRs focused — one feature or fix per PR is easier to review
- Add a short description of any new screen, model, or service you add

## Questions

Open an issue with the `question` label, or reach out at hello@safepathco.app.
